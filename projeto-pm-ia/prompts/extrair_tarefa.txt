## TAREFA
Analise a mensagem do usuário e extraia as informações de tarefa.

## REGRAS DE EXTRAÇÃO

### Responsável (assignee)
- Se mencionar nome de pessoa, faça matching fuzzy com a lista de membros
- Se não mencionar ninguém, deixe null (será atribuída ao remetente)
- Aceite variações: "pro João", "pedir ao João", "João precisa", "mandar pro João"

### Prazo (due_date)
- Converta para ISO 8601 (YYYY-MM-DDTHH:mm:ss)
- "hoje" = data atual às 18:00
- "amanhã" = próximo dia às 18:00
- "sexta" = próxima sexta-feira às 18:00
- "semana que vem" = próxima segunda às 18:00
- Se não mencionar, deixe null

### Prioridade (priority)
- "urgente", "ASAP", "pra ontem" = "urgent"
- "importante", "prioritário" = "high"
- Padrão = "normal"
- "quando puder", "sem pressa" = "low"

### Título (title)
- Extraia a essência da tarefa de forma clara e concisa
- Máximo 100 caracteres
- Não inclua nome do responsável ou prazo no título

### Projeto (project)
- Se mencionar nome de projeto existente, extraia
- Caso contrário, deixe null

## FORMATO DE RESPOSTA
Responda APENAS com JSON válido, sem markdown:

{
  "assignee": "nome exato da lista ou null",
  "title": "descrição clara da tarefa",
  "description": "detalhes adicionais ou null",
  "due_date": "2024-01-15T18:00:00 ou null",
  "priority": "low|normal|high|urgent",
  "project": "nome do projeto ou null",
  "confidence": 0-100,
  "needs_confirmation": true/false,
  "confirmation_message": "pergunta de confirmação se necessário"
}
